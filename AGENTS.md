# Swell Quant — 仓库指南

## 项目定位

Swell Quant 是一个独立的 A 股日频量化研究系统（Python 库形态），不做自动交易、不接券商实盘、不输出投资建议。

## 默认约束

- 市场范围：只支持 A 股日频。
- 产品形态：Python 研究库（marketdata / factors / portfolio 三层），无自动下单。
- 项目边界：不要把本项目代码写入 `swell-lobster`；后续通过 MCP 或 HTTP API 集成。
- 数据边界：所有原始数据、处理数据、产物都放在本项目数据目录下。

## 注释要求

新增代码时，先判断逻辑是否属于关键路径、兼容路径、状态机、路由分发、接口适配、异常回退、数据清洗、因子计算、标签生成、训练切分、回测撮合或业务流程编排。

对上述高重要性逻辑，默认补充简洁中文注释，优先解释为什么这样做、依赖什么前提、失败时如何兜底。尤其是时间序列切分、未来收益标签、复权处理、停牌处理、涨跌停处理、手续费滑点、回测调仓等逻辑，必须写清楚避免未来函数和数据泄漏的关键前提。

对简单样式、纯透传组件、无分支浅包装函数，不强制加注释，避免无信息量注释。

## 金融合规与安全

- 所有用户可见预测和报告必须标注“仅用于研究，不构成投资建议”。
- 不允许新增自动下单、实盘交易、券商登录、资金账户操作能力，除非后续有新的明确计划和安全评审。
- 不允许把 API key、数据源 token、账号密码写入代码或文档正文。
- 不允许把回测收益描述成可实现收益。

## 质量门禁

项目进入代码阶段后，至少需要建立以下校验：

- 后端类型检查和测试。
- 因子与 as_of/PIT 查询的无未来函数测试。
- 回测结果的确定性测试。
