# Swell Quant MVP 计划

## 目标

第一版建设一个 A 股日频研究看板，完成从数据采集、因子工程、模型训练、预测排名、回测评估到 AI 报告的闭环。

项目不做自动交易，不接券商实盘，不提供买卖建议。所有结果仅用于个人研究。

## 范围

v1 做：

- A 股日频行情数据。
- 基础技术因子。
- 未来 5 日收益率和跑赢基准标签。
- LightGBM 模型。
- Top N 组合回测。
- Web 研究看板。
- AI 研究报告。

v1 不做：

- 分钟级、高频、期货、期权、加密货币。
- 实盘交易、自动下单、券商接口。
- 深度学习、强化学习、复杂组合优化。
- 直接用 LLM 判断“买入/卖出”。

## 阶段 0：项目初始化

- 新建独立仓库 `swell-quant`。
- 初始化后端、前端、文档和数据目录。
- 建立 `.env.example`，预留 `DEEPSEEK_API_KEY`、`OPENAI_API_KEY`、`DATA_DIR` 等配置。
- 在 README 和前端关键页面加入“仅用于研究，不构成投资建议”。

验收：

- 项目目录清晰。
- 文档能说明项目定位和 v1 范围。
- 没有安装无关依赖，没有写入密钥。

## 阶段 1：数据采集与存储

- 使用 AKShare 拉取 A 股日频行情。
- 支持按股票、日期区间增量更新。
- 使用 DuckDB 存储原始行情和清洗后行情。
- 明确 v1 数据口径：默认使用前复权日频价格，保留原始 OHLCV、复权因子和数据源更新时间。
- 明确 v1 股票池：先使用可稳定获取的 A 股普通股票，记录 ST、新股、退市、长期停牌和数据缺口股票的纳入或剔除规则。
- 明确 v1 基准：默认使用沪深 300 或中证全指中的一个作为跑赢基准，并在所有标签、回测和报告中保持一致。
- 记录每次采集任务状态、时间范围、成功数量和失败原因。
- 做基础数据质量检查：重复日期、缺失价格、异常成交量、停牌、复权字段缺失。

验收：

- 能拉取一批 A 股历史日频数据。
- 同一日期重复采集不会重复写入。
- 数据质量问题有可读日志。
- 复权、股票池、基准和异常股票处理策略写入数据元数据，后续因子、模型和回测读取同一份口径。

## 阶段 2：因子工程与标签

- 计算基础因子：收益率、动量、均线、波动率、成交量变化、换手率、RSI、MACD。
- 生成未来 5 日收益率标签，默认口径为 T+1 到 T+5 的持有期收益，T 日只作为特征观察日。
- 生成未来 5 日是否跑赢基准的二分类标签，基准收益使用同一持有窗口计算。
- 严格按 `symbol/date` 排序和滚动窗口计算，特征只允许使用 T 日收盘后已经可见的数据。
- 对缺失值、停牌、涨跌停和股票池变更做显式标记，训练前不得用全样本统计量做填充或标准化。
- 保存因子版本、字段列表和生成时间。

验收：

- 因子表按 `symbol/date` 唯一。
- 特征生成不读取 T 日之后的数据，标签只作为监督目标或离线评估使用。
- 缺失数据和停牌数据有明确处理策略。
- 至少有一组无未来函数测试，覆盖滚动窗口、标签错位和全样本统计量泄漏。

## 阶段 2.5：离线研究内核验收

- 串通最小离线链路：数据样本、因子表、标签表、训练样本、一次模型训练和一次回测。
- 固化一份小规模样例数据或可重复生成的测试数据，用于验证核心流程不依赖前端和 LLM。
- 输出一份离线研究报告，包含数据范围、特征版本、训练区间、测试区间、回测参数和风险提示。

验收：

- 一条命令或脚本能复现离线研究内核结果。
- 无 LLM API key、无前端服务时，数据、因子、训练和回测仍可独立运行。
- 样例结果必须标注“仅用于研究，不构成投资建议”。

## 阶段 3：模型训练与预测

- 使用 LightGBM 训练回归或排序模型。
- 按时间切分 train/valid/test，禁止随机切分。
- 对未来 5 日标签采用按日期分组切分，并在 train/valid/test 边界预留至少 5 个交易日 gap，避免标签窗口重叠造成泄漏。
- 支持 walk-forward 评估，记录每个训练窗口、验证窗口、测试窗口和对应模型版本。
- 保存模型文件、训练参数、特征列表和评估指标。
- 输出每个交易日的股票预测分数和 Top N 排名。
- 记录模型版本，支持查看最近一次训练结果。

验收：

- 能训练出一个可复现的 LightGBM 模型。
- 能基于最近交易日生成预测排名。
- 训练、验证、测试时间区间清晰可查。
- 模型训练固定随机种子，重复运行关键指标和预测排序在允许误差内稳定。

## 阶段 4：回测评估

- 实现 Top N 股票组合回测。
- 支持调仓周期、Top N 数量、手续费、滑点参数。
- 明确信号和成交时点：默认 T 日收盘后生成预测，最早 T+1 按指定成交价执行，不能使用成交日之后的数据。
- 处理停牌、涨跌停、股票池不足、无法买入、无法卖出等基础异常，并记录每次调仓失败原因。
- 明确成交价口径，v1 默认支持开盘价、收盘价或 VWAP 近似中的一种，所有结果展示时同步披露。
- 输出净值曲线、年化收益、最大回撤、夏普、胜率、换手率和基准对比。
- 保存每次回测的参数和结果。

验收：

- 能完成一次带手续费的历史回测。
- 回测结果可复现。
- 回测报告明确提示历史收益不代表未来表现。
- 至少有一组回测确定性测试，覆盖调仓、手续费、滑点、停牌和涨跌停处理。

## 阶段 5：前端研究看板

- 首页展示数据更新时间、模型版本、最近回测摘要。
- 预测页展示 Top N 股票、预测分数、关键因子。
- 回测页展示净值曲线、回撤曲线、收益指标。
- 单股页展示 K 线、因子变化、历史预测命中情况。
- 报告页展示 AI 生成的研究说明和风险提示。

验收：

- 无数据、加载中、失败状态都有明确 UI。
- 表格和图表在桌面端可读。
- 所有预测和报告页面都显示研究用途声明。

## 阶段 6：AI 报告与 Agent 集成

- 封装统一 `LLMProvider`。
- 默认使用 DeepSeek V4 Flash 生成低成本摘要。
- 使用 DeepSeek V4 Pro 生成复杂报告。
- 高价值或失败场景可回退 GPT-5.5。
- 报告输入只使用模型输出、回测指标、因子重要性和行情摘要，不让 LLM 直接生成预测分数。
- 后续提供 MCP 或 HTTP API，让 `swell-lobster` 调用更新数据、跑训练、跑回测、生成报告等能力。

验收：

- 无 LLM API key 时，核心预测和回测仍可运行。
- AI 报告失败不影响模型训练和回测。
- 报告不输出确定性投资建议。

## 与 swell-lobster 的关系

`swell-lobster` 是个人 AI 助手运行时，适合作为 Agent 工作台。本项目是垂直量化研究系统，负责数据、训练、回测和预测看板。

推荐集成方式：

- `swell-quant` 暴露 HTTP API 或 MCP server。
- `swell-lobster` 作为调用方，触发研究任务和读取报告。
- 两个项目独立部署、独立数据目录、独立质量门禁。
