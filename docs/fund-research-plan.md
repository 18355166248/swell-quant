# 基金研究助手计划

本计划用于把项目从“股票研究看板”扩展到“基金研究辅助”。当前已落地本地样例基金池、指标计算、候选清单、API 和前端基金页；基金模块仍然只做研究筛选、比较和风险提示，不输出买入、卖出、持仓比例或收益承诺。

## 定位

- 做：基金池筛选、历史风险收益比较、风格暴露说明、费用与规模提示、候选清单。
- 不做：直接告诉用户买哪只基金、预测净值涨跌、给出仓位或买卖点。
- 所有页面和报告继续展示“仅用于研究，不构成投资建议”。

## v1 基金池

第一阶段只覆盖公开可获取的场外基金和 ETF 基础信息，优先选择可稳定获取净值或行情的品种。

- 指数基金和 ETF：适合先做宽基、行业、主题暴露比较。
- 主动权益基金：先只做历史指标和风格描述，不做经理能力断言。
- 货币、债券、QDII：作为后续扩展，避免第一版混入不同风险资产。

## 指标口径

基金候选清单按研究指标拆成四类，避免用单一收益率排序误导。

| 类别 | 指标 | 用途 |
| --- | --- | --- |
| 收益 | 近 1 月、3 月、6 月、1 年收益 | 观察历史表现，不作为未来收益预测 |
| 风险 | 最大回撤、波动率、下行波动 | 识别净值波动和承受难度 |
| 稳定性 | 成立年限、规模、跟踪误差、换手 | 排除样本太短或流动性不足品种 |
| 成本 | 管理费、托管费、申购赎回费 | 评估长期持有成本 |

## 候选评分

第一版使用透明规则，不用 LLM 直接打分。

1. 先按基金类型分组，避免债券基金和股票基金直接横比。
2. 对收益、回撤、波动、费用做分位数标准化。
3. 按用户风险偏好生成不同排序视图：稳健、均衡、进取。
4. 输出“候选理由”和“风险提示”，不用“推荐买入”等措辞。

## 买前验证分层

基金候选清单额外输出 `verification_status`，只表示研究材料是否足以进入人工复核，不表示可以买入。

- `ready` / `可进入人工复核`：指标未触发明显风险，且关键材料齐备。
- `review` / `需补充验证`：存在风险提示、费用/规模/波动问题，或需要补充材料后再判断。
- `block` / `暂不适合决策`：样例数据、关键材料缺失、基金合同/费用/个人风险偏好未复核时使用。

第一版样例基金数据默认会暴露“样例数据不能作为真实申购依据”等阻塞项，避免把工程样例误用为真实基金结论。

## 页面计划

- 基金池页：展示基金代码、名称、类型、规模、成立年限、费用。
- 基金比较页：多基金净值曲线、回撤曲线、风险收益散点图。
- 基金候选页：按风险偏好展示候选清单、指标归因、风险提示和买前验证状态。
- 基金详情页：展示历史收益、回撤、费用、持仓/行业暴露（如数据源可用）。

## 数据与验收

- 已建立 `sample_funds.csv`、`sample_fund_nav.csv`、`sample_fund_metrics.csv` 和三类 `sample_fund_candidates_*.csv` 样例产物，保证页面、指标和测试可复现。
- 已建立 `make fund-trial-dry-run`、`make fund-trial` 和 `make fund-trial-status`，沿用股票真实试跑的预演与摘要机制；摘要写入 `data/reports/fund_trial_run.json`，API 通过 `/api/funds/trial` 展示。
- 真实基金试跑当前只验证公开基金名称和净值序列，基金经理、持仓、行业暴露和费用细项仍作为人工复核阻塞项处理。
- 验收必须覆盖：数据完整性、费用字段缺失处理、风险指标确定性、页面免责声明。

## 风险边界

- 历史收益、回撤和评分不能作为未来收益保证。
- 用户风险偏好必须显式展示，默认不生成任何“买入”动作。
- 基金与股票候选清单分开展示，避免把单股模型预测误用为基金选择依据。
