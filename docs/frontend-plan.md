# 页面开发计划

## 目标

v1 前端是研究看板，不做营销页。首屏直接进入工作台，让用户快速判断数据是否新、模型是否可用、最近回测是否可信，以及 AI 报告是否存在风险提示。

所有预测、排名、回测和报告页面都必须展示“仅用于研究，不构成投资建议”。页面只展示研究结果和任务状态，不承载数据清洗、训练、预测或回测核心逻辑。

## 技术选型

- React + TypeScript + Vite。
- Ant Design 负责布局、表格、表单、按钮、空状态、加载状态和消息提示。
- ECharts 负责净值曲线、回撤曲线、K 线、因子趋势和指标对比。
- React Query 管理服务端状态、轮询和错误重试；局部交互状态使用 React state。
- 前端目录遵循 [架构设计](architecture.md) 中的 `frontend/src/pages`、`components`、`api`、`types` 分层。

## 页面清单

| 页面 | 路由 | 核心用途 | 优先级 |
| --- | --- | --- | --- |
| 首页工作台 | `/` | 汇总数据、模型、预测、回测和报告状态 | P0 |
| 任务中心 | `/tasks` | 查看数据更新、训练、预测、回测、报告生成任务 | P0 |
| 预测页 | `/predictions` | 查看最近交易日 Top N 排名、预测分数和关键因子 | P0 |
| 回测页 | `/backtests` | 查看回测列表、参数、净值曲线、回撤和指标 | P0 |
| 单股页 | `/stocks/:symbol` | 查看单股行情、因子走势和历史预测表现 | P1 |
| 报告页 | `/reports` | 查看 AI 研究报告和结构化风险提示 | P1 |
| 设置页 | `/settings` | 查看数据目录、API 连接状态、LLM key 是否配置 | P2 |

## 全局框架

- 左侧导航：工作台、任务、预测、回测、单股、报告、设置。
- 顶部状态条：数据最后更新时间、当前模型版本、最近回测 ID、研究用途声明。
- 主内容区：页面级标题、筛选条件、核心图表/表格、风险提示。
- 全局状态：无数据、加载中、失败、任务运行中、结果过期。

实现要求：

- 顶部研究用途声明始终可见，不只放在报告页。
- 所有触发型按钮只发起后端任务，不在前端直接计算研究结果。
- 错误提示必须包含任务名、失败阶段和可读原因。

## 页面细化

### 首页工作台

目标：用一屏判断研究链路是否可用。

模块：

- 数据状态：股票池、日期覆盖、最近更新时间、数据质量问题数量。
- 模型状态：最新模型版本、训练区间、验证区间、测试区间、关键指标。
- 最近预测：最近交易日 Top 10、预测分数、生成时间。
- 最近回测：年化收益、最大回撤、夏普、换手率、基准对比。
- 最近报告：报告生成时间、摘要、风险提示数量。

依赖接口：

- `GET /api/data/status`
- `GET /api/models/latest`
- `GET /api/predictions/latest`
- `GET /api/backtests/{id}` 或最新回测摘要接口
- `GET /api/reports/{id}` 或最新报告摘要接口

验收：

- 任一模块无数据时展示明确空状态。
- 数据、模型、回测过期时有明显但克制的状态提示。
- 页面不出现买入、卖出、推荐、稳赚等交易建议措辞。

### 任务中心

目标：让用户看清任务是否正在运行、失败在哪里、产物在哪里。

模块：

- 任务列表：类型、状态、开始时间、结束时间、耗时、输入参数摘要、失败原因。
- 任务详情：日志片段、输出路径、关联模型版本或回测 ID。
- 任务触发：数据更新、训练、生成预测、运行回测、生成报告。

依赖接口：

- `GET /api/tasks`
- `GET /api/tasks/{id}`
- `POST /api/data/update`
- `POST /api/models/train`
- `POST /api/predictions/run`
- `POST /api/backtests/run`
- `POST /api/reports/generate`

验收：

- 运行中任务支持轮询刷新。
- 同类写任务排队或阻塞时，按钮给出明确状态。
- 失败任务展示可读错误，不只展示 HTTP code。

### 预测页

目标：查看最近预测排名和影响预测的主要因素。

模块：

- 筛选：交易日、模型版本、Top N、行业或股票池。
- Top N 表格：排名、代码、名称、预测分数、关键因子、是否停牌/涨跌停标记。
- 分布图：预测分数分布、Top N 与全池对比。
- 风险提示：数据日期、模型版本、基准同源说明。

依赖接口：

- `GET /api/predictions/latest`
- `GET /api/predictions?date=&model_version=&top_n=`

验收：

- 表格默认按预测排名排序。
- 停牌、涨跌停、数据缺口必须有显式标记。
- 页面显示“预测分数来自模型，仅用于研究，不构成投资建议”。

### 回测页

目标：判断策略回测是否可复现、是否受交易约束影响。

模块：

- 回测列表：ID、模型版本、股票池、Top N、调仓周期、手续费、滑点、运行时间。
- 指标卡：累计收益、年化收益、最大回撤、夏普、胜率、换手率、基准收益。
- 图表：净值曲线、回撤曲线、相对基准曲线。
- 调仓明细：日期、买入、卖出、无法成交原因。
- 参数详情：成交价口径、T+1 开盘成交、基准同源说明。

依赖接口：

- `GET /api/backtests`
- `GET /api/backtests/{id}`

验收：

- 所有收益展示为历史回测结果，不描述为可实现收益。
- 回测参数和模型版本可追溯。
- 无法成交、停牌、涨跌停处理在明细中可见。

### 单股页

目标：用于解释单只股票在模型和因子中的历史表现。

模块：

- 股票基础信息：代码、名称、所属股票池、数据覆盖区间。
- K 线图：前复权日线、成交量。
- 因子走势：动量、均线、波动率、成交量变化等。
- 历史预测：预测分数、排名、未来 5 日标签、是否跑赢基准。

依赖接口：

- `GET /api/stocks/{symbol}`
- `GET /api/stocks/{symbol}/prices`
- `GET /api/stocks/{symbol}/features`
- `GET /api/stocks/{symbol}/predictions`

验收：

- 当前暂无接口时可先用空状态占位。
- 历史预测只作为研究解释，不出现交易建议。

### 报告页

目标：展示 LLM 生成的研究报告，同时保留结构化依据。

模块：

- 报告列表：报告 ID、生成时间、关联模型版本、关联回测 ID。
- 报告正文：摘要、模型解释、回测总结、风险提示。
- 结构化依据：数据范围、训练区间、回测参数、因子重要性。
- 降级状态：无 LLM key、LLM 失败、内容校验失败。

依赖接口：

- `GET /api/reports`
- `GET /api/reports/{id}`
- `POST /api/reports/generate`

验收：

- AI 文本失败时仍展示结构化指标。
- 报告必须包含研究用途声明和风险提示。
- 报告不输出确定性买卖建议。

### 设置页

目标：降低本地研究环境排查成本。

模块：

- 数据目录：`DATA_DIR`、DuckDB 路径、产物目录。
- API key 状态：只显示是否配置，不显示真实值。
- 服务状态：后端版本、最近任务状态、前端构建版本。

依赖接口：

- `GET /api/settings`
- `GET /api/health`

验收：

- 不展示任何 secret 明文。
- 配置缺失时给出修复方向。

## 实施顺序

1. 前端工程初始化：Vite、TypeScript、Ant Design、ECharts、React Query、基础路由。
2. 全局框架：导航、顶部状态条、研究用途声明、错误边界。
3. API client 和类型定义：先按 API 草案定义 mock-friendly 的类型。
4. 任务中心：优先打通任务状态，因为后续页面都依赖任务产物。
5. 首页工作台：接入数据、模型、预测、回测和报告摘要。
6. 预测页和回测页：优先做 P0 研究判断能力。
7. 报告页和单股页：补解释能力。
8. 设置页：补环境排查能力。

## Mock 与联调策略

- 前端第一版可以使用本地 mock 数据开发页面结构，但 mock 字段必须与后端 API 类型一致。
- mock 数据必须覆盖空状态、加载中、失败、运行中、成功、有数据质量问题等状态。
- 后端接口可用后逐页替换 mock，不做大批量一次性切换。
- 每个页面至少保留一个无数据快照和一个正常数据快照用于人工验收。

## 页面质量门禁

- `npm run build` 或等价构建检查通过。
- 关键表格支持排序、分页或限制 Top N，避免一次渲染过大数据。
- 图表必须显示单位、时间范围和数据来源。
- 所有页面在 1366px 宽桌面端可读；v1 暂不优先优化移动端。
- 用户可见预测、回测、报告都展示“仅用于研究，不构成投资建议”。
