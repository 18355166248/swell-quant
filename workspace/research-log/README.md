# research-log/ — 研究日志

一次研究会话一个文件：`YYYY-MM-DD-<主题>.md`。

必须记录（防 p-hacking 的关键是**全量登记**）：

- **动机与假设**：为什么认为这个因子/改动可能有效（理论支撑）。
- **方法与参数**：因子定义、池、区间、step/horizon、成本、基准——足以复现。
- **全部尝试**：本次试过的所有因子/参数组合及其结果，包括无效的。
- **中间观察**：意外现象、数据问题、口径疑点。
- **下一步动作**。

结论达到 [preferences.md](../preferences.md) 的三条闸门后，摘要升入
[findings/](../findings/)，并在日志中注明。
